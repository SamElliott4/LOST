LOST API interface overview

lost_key
Returns the LOST public key for encryption of other requests
example
Request:
No input needed, but arguments and signature fields must be present for the request
Response:
result 'OK' or 'FAIL'
key LOST public key

activate_user
Reinstates LOST access for an existing user, or creates a new user account if non exists.
example
Request:
username OSNAP username to activate
Response:
result 'OK' or 'FAIL'

suspend_user
Revoke access for user.
This will have no effect if the user does not exist, or access has already been revoked.
example
Request:
username OSNAP username to suspend
Response:
result 'OK' or 'FAIL'

list_products
Returns a listing of LOST products based on filter criteria.
example
Request:
vendor Case-insensitive vendor name
description Case-insensitive description of product
compartments Secutiry tags as json array; must match for compartment and level
Response:
listing A json list of matching product information

add_products
Adds products to LOST. Requests are atomic (i.e. either all new products are added or none are.)
Attempting to add a duplicate product will cause the request to fail.
example
Request:
new_products A json list describing the new products
Response:
result 'OK' or 'FAIL'

add_asset
Add a new asset to LOST
example
Request:
vendor Vendor of the product
description Description of the asset
compartments A json list of additional compartments the asset should be listed under
facility Facility code the asset is located at
Response:
result 'OK' or 'FAIL'
